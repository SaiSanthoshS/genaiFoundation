Automated Code Documentation Generator
Agent: When user submits a repository URL and branch, the agent fetches source files, parses the code structure to identify every undocumented function, generates clear docstrings for each one, detects REST endpoints and produces an OpenAPI specification, then opens a pull request with all changes.
Frontend: Repo URL & Branch Input — submission triggers the agent | File Explorer — files colour-coded by the agent: green (documented), red (undocumented) | Documentation Preview — side-by-side before/after diff per function, generated by the agent | API Spec Viewer — Swagger UI embed populated with the agent's OpenAPI output | Pull Request Summary — agent-written PR description; user clicks "Open PR" to submit
